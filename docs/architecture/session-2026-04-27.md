### Three-Layer Verification Pipeline (10% → 90%)

- **Layer 1: Lint guard** — `discoverLintCommands()` per runtime (go vet, ruff, flake8, eslint, clippy). Pre/post-edit lint comparison with new-error diffing. New lint errors → critical block
- **Layer 2: Test execution** — Exit codes captured from `exec.CommandContext` via `exitCodeFromError()`. `ToolResult.ExitCode` and `HostObservation.ExitCode` populated
- **Layer 3: LLM judge** — `POST /v1/review/change` with pluggable `ReviewFunc`. Reviewer model (different from implementer) evaluates diff against task objective. Structured verdict: APPROVE/CHANGES_REQUESTED/REJECT
- All three feed into `decide_approval_policy()` in Python
